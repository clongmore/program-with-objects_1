STAGE 0: The Basics of the Web and HTML

Important Concepts

The Internet is a structure made up of millions of computers whose users can comunicate with each other and share information.

The World Wide Web (WWW) gives Internet users easy access to information on the Internet.

HTML is Hypertext Markup Language and it is the main, or most common, type of document on the web. There are other types such as text, images, videos, and music. HTML was designed to describe the contents of a web page in a very general way.

HTTP is Hypertext Transfer Protocol that sends requests to a server.

Computers are stupid because they can only do what programmers tell them to do. The programmers have to write in a language that the computer can read. This is referred to as CODING.

Elements and Tags

  • ELEMENT: Everything within a set of opening and closing tags.
  • TAG: Turns on a feature which is indicated by an opening tag, and turns off a feature indicated by a closing tag.
  • INLINE ELEMENTS: Tags that do not require a closing tag.
  • ELEMENTS: Tags that require a closing tag. Block Elements are similar to an invisible box being placed around the text.
  • ATTRIBUTE: A property of an HTML element.
  • <b>gives you BOLD text.
  • <em> adds emphasis such as italics.
There are many others. There are tags to do almost everything. After all, computers are stupid and will only do what we tell them to do, so we have to talk to them in a language they can understand--HTML

STAGE 1: Creating Structured Documents with HTML

Markup Languages and Structure

The structure of a web page can be determined by looking at its coding or markup language.

HTML (Hypertext Markup Language) is the main, or most widely used, programming language. It controls the structure of a web page (how it is set up).

CSS (Cascading Style Sheet) is a programming language used to make the structure look like the design you were provided. CSS controls the style of the HTML elements, or the way the website will look to the user. The instructor gave a good example of cascading by using the laws of the land. There are national laws that govern the whole nation. From the national laws there are state laws (what you can do in a state), and from there we have city laws, county laws, and house laws. In CSS, the most specific rules are applied to every element. For example, the most specific rule is the house rules so it would be the same as the national laws through cascading. Cascading keeps going down to levels until you get to the most specific rule and that rule determines the visible style for the page.

DOM (Document Object Model) is the tree-like structure of a page. This structure has a series of nested boxes, which means there are boxes within boxes, within boxes. I can remember this concept if I think about my family tree, which is rather large. My grandmother had nine children, my mother had nine children, and all of my siblings had children, and their children had children. Nested elements are indented, so the more indented they are the more deeply nested the boxes are.

DIV Tags <div> can define boxes and help to divide other HTML elements into sections that we can treat as one whole group. Give each <div> tag a class element to better define the tag. An example would be a class attribute "TOC" which defines the Table of Contents for the page.

The Benefits of Avoiding Repetition

Repetition is doing the same thing over and over again, as in copying/pasting. The code is more prone to errors if, for instance, the code that was copied/pastied has to be changed. If the change isn't made in every place where it was copied/pasted, errors can arise. Programmers should avoid repetition at all costs, and one way to avoid it is by using Inherit. This is one of the key features in CSS.

With Inheritance properties are applied to a specified element as well as its descendants. Descendant elements inherit text-related properties but not box-related properties. Properties that can be inherited are color, font, letter-spacing, line height, list-style, text-alig, text-indent, text transform, visibility, white-space, and word spacing. Inheritance allows programmers to write less CSS by preventing certain properties from being repeated over and over again.

5 Ways of Thinking Like a Programmer

We have to use these all the time.

  1. Procedure Thinking: Creating very clear and explicit instructions for the computer to follow.
  2. Abstract Thinking: Finding generality (similarity) amongst things that seem to be different.
  3. Systems Thinking: Breaking big problems into small pieces.
  4. Technological Empathy: An understanding of how tools and systems work.
  5. Debugging: Systematically identifying the cause of why something isn't working.

STAGE 2 WORK SESSION 2: Variables and Strings

What is a Variable

Variables are used by programmers to assign value to a name or a piece of information. As the word implies, variables can vary and you can change the values of a variable.

Assigning a Value to a Variable

In order to use a variable, you must first assign a value to the variable or name. As we covered in our lesson, often times names are used to keep up with variables that are big numbers. You assign a value to the variable by using an Assignment Statement.

The assignment statement is Name = Expression

Name is the value it refers to. Give it a name that will help you remember what the value of the expression is. During the assigning the Name must start with a pound sign (#) and you must use an underscore (_) to separate the words in the name.

Expression is something like a number. Any value is an expression.

This is an example of an assignment statement for the speed of light.

#speed_of_light
speed_of_light=299792458
print speed_of_light

When I click Run, the computer will return the value of the speed of light, 299792458.

So in this example I did the following:

  • I assigned the Name: #speed_of_light
  • I gave the Expression a value: speed_of_light=299792458
  • I could then run the statement: print speed_of_light

The Difference Between Equals (=) in Math Verses In Programming

In Python, what we may consider to be math commands are actually assignments in Python. "

Take for example the = sign. In math this is an equal sign. In Python it means assignment. Think of it as <==. An example using the assigment statement Name=Expression would be:
#hours
hours=9
print hours

Another example would be the + sign. There is a difference in the math command "2+3=5". As noted above, this would be considered an assignment in Python. Let's define it using the asignment statement Name=Expression.
#my_variable
my_variable=5
print my_variable

How Are Variables Useful

Variables play an important role in computer programming because they enable programmers to write flexible programs.

Variables save time when entering code. Instead of entering data directly into a program, a programmer can use variables to represent the data. Then, when the program is executed, the variables are replaced with real data. This makes it possible for the same program to process different sets of data.

Variables are useful when writing programs that deal with big numbers.

The opposite of a variable is a constant. Constants are values that never change. Because of their inflexibility, constants are used less often than variables in programming.

Working With Strings

Strings are a sequence of characters surrounded by "quotes", either single quote or double quotes.

You must end with the type of quote you started with. If you start with single quote as in 'I am stringful' you must end with a single quote. You cannot mix the quotes.

Without the quotes the computer sees it as a variable and an error occurs.

Concatenation

Concatenation is connecting a series of strings. The strings are connected using the "+" operator. An example would be:

name="Carolyn"
print "Hello" + name

The output is: Hello Carolyn

Indexing Strings

It is possible to extract subsequences from a string if it does not include numbers.

Characters of a string are 0 1 2 3 4 5 6 etc. To select subsequences you need a string and an expression.

Following is an example of extracting subsequences from a string.
name="Carolyn"
print name[0]
Run returns the character 'C' because 'C' is in position 0.

name="Carolyn"
print name[4]
Run returns the character 'l' because 'l' is in position 4.

Remember the first position is position 0.

Selecting Subsequences from Strings

It is possible to select subsequences from strings.

A string that is a subsequence of the characters in a string start from position "start" and end with position "stop" but not including the character at "stop". For the expression you have a "start" number and a "stop" number. The start expression and stop expression are separated by a colon ':'. Therefore the subsequence is start + stop minus 1.

Finding Strings in Strings

Using find with the string gives us the first position where the target string is found. If the string is not found anywhere then it returns a '-1'.

Find has two strings. The second string follows the word find and is surrounded by parenthesis at the beginning and the end. What goes inside the parenthesis of a find string is called a Parameter.

STAGE 2 WORK SESSION 3: Input-Function-Output

What Is a Function

In the lesson Dave used the word procedure in the place of function, but they are the same. A function is a procedure that takes input, does something with it, and then produces output. The number of inputs has to match the number of outputs the procedure, or function, expects. Inputs are also called operands or arguments.

The Difference Between Making and Using a Function

Making a function is writing a code to accomplish a specific task. We make a function or procedure by using def followed by the procedure. For example, to make a procedure to find the sum of two numbers I would define it by using def, as in def sum(a,b). I expect the output to return (a+b), so I am going to write a return statement, return a+b. Now, to use the function I am going to print sum(2,3).

How Do Functions Help Programmers Avoid Repetition

Once a function is made, it can be used over and over again to produce output, which saves time typing.

We can also use what is called procedure composition where we use the output of one procedure as the input of another procedure.

What Happens If a Function Doesn't Have a Return Statement

All functions require a return statement. The return statement tells the computer what you expect it to produce when it executes the function. If there is no return statement, when the procedure is run it will return the output None.

STAGE 2 WORK SESSION 4: Decisions and Repetitions: If and While

Thinking Procedurally - Making Decisions

Thinking as a programmer includes thinking procedurally and making decisions. Making decisions deals with doing comparisons and noting the likenesses or differences of objects.

Boolean Value: True or False

Python uses a lot of operators for doing comparisons. The output of a comparison is not a number but a Boolean Value. A Boolean Value is one of two things: True or False.

Some comparison values are: Less than (<), Greater than (>), Not equal to (!=), and equality comparisons (==).

Why is the Equality Comparison Done Using Double Equal Signs (==) Instead of Single Equal Signs (=)

The (=) sign means assignment. It assigns a value to an object. For example, I=21 assigns a value to I.

Using (==) does a comparison. For example, I==21 does a comparison that is either True or False depending on whether the value of I is 21.

Using If Statements

To make a code do something different depending on the result of the comparison is to use if statements.

A if statement is written as:

if <Test Expression>:
<Block>

The <Block> executes if True. It does not execute if False.

While Loops

While Loops let you write code that does the same thing over and over.

Similar to the if, a while loop statement is written as:

while <Test Expression>:
<Block>

If the test expression is True, the <Block> executes and it keeps going back to the test expression. Once the test expression is False, it proceeds to the next instruction.

An example of a while statement is:

I=0
while I <10:
print I
I = I+1

If a loops keep going forever, it is called an infinite loop.

Break stops the loop even while the statement is true.

Debugging

Bugs are inevitable. They are going to happen. The important thing is not to avoid them entirely but to develop a strategy to systematically get rid of them.

When Python code crashes it give you a Traceback. The Traceback tells you what line it crashed and the function name. This helps you to focus your attention on where the error occurred. Given the error, you can use a search engine such as Google to find the fix.

Debugging isn't just correcting errors or figuring out how you made a mistake. It is a scientific process of examining a code to discover things you did not expect in it.

Bugs can cause problems that are a lot bigger than getting a weird error message on your screen. Debugging is a big part of writing reliable software.

STAGE 2 WORK SESSION 5: Structured Data: Lists and For Loops

Structured Data

Structured data includes Strings and Lists.

A string is a sequence of characters such as S = 'yabba!'

A list, however, is a sequence of anything. For example: p=['y','a','b','c','!'].

A nested list is a l.ist within a list. It can be strings, or numbers, or both. Nested lists can be two lines but you have to be sure to separate the lines after a comma so that the interpreter will know it is one list and not two. The first element of a nested list starts at 0.

Mutability

Lists support mutation. Mutation means we can change the value of a list after we have created it.

What happens with strings:

  1. We can create a string. For example: s='Hello'
  2. We can store it in a variable. The variable for the string s='Hello' is the "s".
  3. We can change the value of a variable by creating a new string. For example: s='Yello'
  4. We changed the value of "s" but it doesn't change the value of the string. It creates a new string.
  5. We can use the (+) operator to concatenate two strings. For example: s=s + 'w'
  6. With concatenation we do not modify the string we had. Using the (+) operator creates a completely new string where the value of the new string is the result of concatenating the two input strings. Consider s='Yello' + 'w'. The result is s='Yellow'.

Aliasing

Aliasing is when we have two different ways to refer to the same object.

If we have two variable names that refer to the same object, any changes we make to one of the objects that the variable refers to will also affect the value of the object that the second variable refers to.

To keep from affecting both objects:

  1. Create a new object.
  2. Assign one of the variables to the new object.
  3. Now, since one of the variables is assigned to the new object, the value of the other object is not affected when changes are made.

This is Technological empathy. We are learning how technology actually works. This empathy will help in diagnosing and fixing bugs that would otherwise be a mystery to us.

The Difference Between Append and Plus (+)

Append is another List Operation. It is a method similar to the find procedure. It is mutating the old list and adding a new element object to the end of a list. This adding is not the arithmetic plus (+). This adding can actually be referred to as affixing because we are affixing an element at the end of a list to store complex data.

Len is another List Operation that is short for length. Here is an example of finding the value of "len" after running the following code:

  • p=[1,2]
  • p.append(3)
  • p=p+[4,5]
  • Doing append the list will be: [1,2,3,4,5]
  • len(p) comes out to be "5" when the code is run.

Object-Oriented Programming: OOP

Let us define or give an overview of object-oriented programming, or OOP, as we get started.

I have a confession to make. When we began writing code and saving the files on our C-drive in a folder that we created called OOP, it never dawned on me that there was something significant as to why we created the file as OOP. Imagine a light bulk shining brightly over my head when I realized that the OOP folder name that I had been saving files into was short for Object-Oriented Programming. Understanding this made the upcoming concepts much easier to comprehend.

Object-oriented programming (OOP) is a programming language model organized around objects and data rather than actions and logic, as is the case with HTML. With OOP what we really care about are the objects we want to manipulate rather than the logic required to manipulate them.

OOP makes it easy to maintain and modify existing code. New objects can be created with small differences to existing ones. It is a useful way to organize information in a conceptual way. It enables us to represent our concept for large projects in an object, thereby allowing us the freedom to manipulate our objects in a concise and efficient way.

An object oriented program is based on classes and there exists a collection of interacting objects, as opposed to the conventional model, in which a program consists of functions and routines. In OOP, each object can receive messages, process data, and send messages to other objects.

To demonstrate what OOP is we will take for an example baking a cake:

  • A class definition can be compared to the recipe to bake a cake.
  • A recipe is needed to bake a cake.
  • The main difference between a recipe (class) and a cake (an instance or an object of this class) is obvious.
  • A cake can be eaten when it is baked, but you can't eat a recipe, unless you like the taste of printed paper.
  • Like baking a cake, an OOP program constructs objects according to the class definitions of the program.
  • A class contains variables and methods. If you bake a cake you need ingredients and instructions to bake the cake.
  • Accordingly, a class needs variables and methods. There are class variables which have the same value in all methods.
  • And there are instance variables, which normally have different values for different objects.
  • A class also has to define all the necessary methods which are needed to access the data.

Another example would be a paper pattern and shirts. In this case the paper pattern corresponds to a class and the shirts are instances or objects of this class.

Another advantage to OOP is abstraction, which is the hiding of detail behind documentation. Abstraction allows us to focus on the program that we want to write and use their functions simply by reading their documentation.

    • In OOP, a class is a template, similar to a blueprint, that puts functions and variables together really well to describe how to create user-defined objects.
  4. HTML and OOP use Inheritance
    • Properties are applied to a specified element as well as its decendants.
    • In HTML, decentant elements inherit text-related properties but not box-related properties.
    • In OOP, inheritance is used to indicate that one class will get most or all of its features from a parent class. For example, the parent's last name, eye color, and skin color, can be inherited by the child.

A class has both data and methods. Just as a blueprint of a building can create several types of buildings, a class can make multiple instances, or objects.
